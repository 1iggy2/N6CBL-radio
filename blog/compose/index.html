Write once. Commit source.

This owner-only tool submits structured blog source and staged images to GitHub through the Cloudflare Worker. The build workflow regenerates the public static HTML after the source commit lands.

One-time repository / Cloudflare setup Required before submit works
  1. Cloudflare Access must protect both /blog/compose/* and /api/blog/publish. The Access policy should allow the owner email address that is allowed to publish.
  2. wrangler.jsonc carries the non-secret Worker var GITHUB_REPO. If this repository is renamed or forked, update that value before deploy.
  3. Add the required Worker secret with npx wrangler secret put GITHUB_TOKEN. Use a GitHub token that can write repository contents so the Worker can create the post JSON plus any images/blog/YYYY-MM-DD-slug/ uploads.
  4. Optional tighter gate: add ALLOWED_EMAILS with npx wrangler secret put ALLOWED_EMAILS. Use a comma-separated list of Access-authenticated email addresses. Legacy ALLOWED_EMAIL still works for one address.
  5. GitHub Actions deployment still needs repository secrets CLOUDFLARE_API_TOKEN and CLOUDFLARE_ACCOUNT_ID so pushes to main can publish the Worker/site.
