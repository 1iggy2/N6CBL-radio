Welcome / how to post

TYPE: SITE NOTE TAGS: workflow, photos, field-journal STATUS: LIVE

Welcome to the N6CBL.radio field journal. This page is a static, human-maintained log for activation reports, station changes, project notes, useful failures, and photos that deserve a longer shelf life than a social-media post. The index above is the short table of records; the full text lives below it in newest-first order.

To make a new post, open /blog/compose/ and fill in the date, title, type, tags, index context line, body paragraphs, and any photo captions. The composer runs entirely in the browser. It does not publish anything by itself; it only produces the exact table row and article block to paste into this file.

If the post has images, first resize and compress them. Prefer WebP; JPEG is fine for already-compressed camera exports. Upload the files under /images/blog/YYYY-MM-DD-slug/, where the slug matches the post title closely enough that a future reader can identify the owner. Every image should have useful alt text and a caption that says what the reader is seeing, where it happened, and why it matters.

In GitHub, edit /blog/index.html. Paste the generated table row at the top of the Post index table body, then paste the generated <article> block directly under the NEW POST INSERTION POINT comment. Keep dates in YYYY-MM-DD format, keep newest posts first in both places, and update the post count in the /blog/ — field journal metadata line.

Before committing, check the link in the new table row points to the new article id, image paths resolve, and the post still reads like a field note: concrete date, place, gear, frequencies or bands, operating conditions, what broke, what worked, and what should be remembered next time. Commit directly with an imperative message such as Add Crystal Cove field report; Cloudflare publishes the committed static files.
