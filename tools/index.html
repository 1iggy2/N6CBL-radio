/tools/ — workbench 17 utilities / static / client-side
Workbench dossier
Purpose Small browser utilities for the questions an operator actually asks on the bench, in the field, and during planning.
Delivery Static HTML/CSS with vanilla JS. No build step. No telemetry. No external scripts. No cookies. No accounts.
Trust boundary Every tool runs in your browser. File parsers (ADIF, Cabrillo) never upload anything. Reference data (band plans, CTCSS tones, exam pools) is committed in this repo.
Scope rule A tool earns a slot when it answers a recurring radio question faster than a calculator app or a search engine, and stays useful offline.

Design rules commitments
Rule Reason Effect
No external scripts A radio tool must keep working on a tethered phone in the field, with a flaky cell signal and the operator's hands on the radio. Every dependency ships from this origin.
No uploads ADIF logs and Cabrillo exports contain personal contact data. A static page has nothing to upload to. All file parsing runs in the browser.
Show the math A calculator that only prints an answer cannot be checked. Reference tables and unit toggles let the operator verify the result. Each tool exposes inputs, formulas, and reference tables.
Mobile-first density Field use happens on a phone. Tools must fit a 390 px viewport without horizontal scroll or icon-only controls. Dense labeled tables, no decorative chrome.
